Redesign this React application to look like it was built by an experienced frontend developer rather than generated by AI.

Goals:
- Make the UI minimal, clean, modern, and professional.
- Prioritize usability over visual effects.
- Remove anything that feels like a marketing landing page.
- Follow good UI/UX principles used in real SaaS dashboards and developer tools.

Design Requirements:

1. Remove unnecessary sections including:
   - Hero section
   - Workflow explanation
   - About CNN section
   - Architecture section
   - Model information cards
   - Large feature cards
   - Excessive statistics dashboard
   - Decorative backgrounds
   - Toast animations unless actually needed

2. Keep only the essential interface:
   - Navbar
   - Application title
   - Short one-line description
   - Large review text area
   - Analyze button
   - Prediction result
   - Confidence score
   - Probability bar
   - Footer

3. Layout
   - Single-page centered layout
   - Maximum width around 900px
   - Plenty of whitespace
   - Simple spacing using Tailwind
   - Responsive without excessive breakpoints

4. Styling
   - White or light gray background
   - Minimal accent color (blue or indigo)
   - Rounded corners (rounded-xl)
   - Soft shadows only where needed
   - No glassmorphism
   - No neon colors
   - No glowing effects
   - No animated gradients

5. Typography
   - Use Inter font
   - Clear visual hierarchy
   - Simple headings
   - Professional sizing
   - Avoid oversized text

6. Animations
   - Remove unnecessary Framer Motion animations.
   - Only keep subtle fade or scale animations on results.
   - No floating elements.
   - No continuous animations.

7. Result Card
   Display:
   - Positive/Negative badge
   - Confidence percentage
   - Horizontal probability bars
   - Short explanation:
     "The model predicts this review is Positive with 94% confidence."

8. Input
   - Large textarea
   - Character counter
   - Sample review button
   - Clear button
   - Analyze button

9. Colors
   Primary:
   - Indigo
   - Slate
   - Gray

   Positive:
   - Green

   Negative:
   - Red

10. Code Quality
    - Remove unused components completely.
    - Remove unused imports.
    - Simplify App.jsx.
    - Avoid deeply nested components.
    - Keep component structure clean.

11. Overall Feel
This should look like a polished machine learning application built by a software engineering student—not an AI-generated landing page.

Think:
- GitHub open-source project
- Kaggle demo
- Hugging Face Spaces interface
- Streamlit-style simplicity with React polish

Do not add any new sections unless they improve usability. Simplicity is preferred over visual complexity.
